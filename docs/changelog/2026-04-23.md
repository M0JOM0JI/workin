## 변경사항

### ✨ 새 기능 (Features)

**API (NestJS)**
- `Store` 모델 필드 확장: `businessOwner`, `businessNumber`, `phone`, `mobilePhone` 컬럼 추가
- Prisma 마이그레이션: `20260423_add_store_business_fields`
- `CreateStoreDto` / `UpdateStoreDto` 확장 필드 반영

**웹 대시보드 (Next.js)**
- **온보딩 페이지** `/onboarding/store`: 첫 매장 등록 전용 페이지
  - 매장명·사업자명·사업자번호(필수) + 주소·전화·휴대폰·업종(선택) 7개 필드
  - 필수 항목 클라이언트 유효성 검사
  - 생성 후 `currentStoreId` 설정 → 대시보드 이동 (`router.replace`)
- **로그인·가입 후 리다이렉트** 개선: `stores.length === 0`이면 `/onboarding/store`로 이동
- **사이드바 매장 추가 모달** 확장: 기존 3개 필드 → 전체 7개 필드
- **매장 설정 페이지** 확장: 사업자명·사업자번호·매장전화번호·담당자 휴대폰번호 필드 추가

### 🏗️ 인프라 (Infra)
- 로컬 `postgresql-x64-16` Windows 서비스 중지·비활성화 (Docker 컨테이너와 포트 충돌 해소)
- `apps/api/.env`: `localhost` → `127.0.0.1` (IPv6 resolve 문제 해소)

### 📝 문서 (Docs)
- `CLAUDE.md`: 브랜치 전략 섹션 추가 (`feature/<작업명>` 네이밍 규칙)
- `CHECKLIST.md`: 완료 항목 상태 업데이트 (스키마 확장, 온보딩, 설정 페이지)
- `docs/PROGRESS.md`: W-03 온보딩 0%→90%, W-10 설정 60%→100%, 전체 79%→86%

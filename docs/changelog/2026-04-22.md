# 2026-04-22 배포

## 배포 요약
Workin 알바 관리 서비스 초기 버전 배포 — 모노레포 전체 구조, API/웹/모바일 기반 기능, Docker 환경, GitHub 동기화 완료

## 변경사항 (1차)

### ✨ 새 기능 (Features)

**API (NestJS · port 3000)**
- Auth: 회원가입 / 로그인 / 토큰 갱신 / 내 정보 조회 (JWT, bcrypt)
- Stores: 매장 CRUD, 직원 목록, 초대 코드 발급·검증 (Redis TTL 24h)
- Schedules: 스케줄 CRUD (날짜 범위 쿼리)
- Attendance: 출근·퇴근 체크, 출퇴근 내역 조회 (중복 출근 방지)
- Payroll: 급여 요약 조회 (year/month 파라미터)
- 공통: Swagger(`/api/docs`), ValidationPipe, JwtAuthGuard, RolesGuard, `@CurrentUser` 데코레이터

**웹 대시보드 (Next.js · port 3001)**
- 로그인 페이지 (Zustand 인증 스토어 persist)
- 대시보드: 실시간 출근현황, 오늘 스케줄, 급여 요약
- 스케줄: 주간 그리드 (실제 API 연동)
- 알바생 목록: 재직/퇴직 필터
- 출근현황: 30초 자동 갱신
- 급여: 월 선택, summary 형태
- 매장 설정: 매장 수정/삭제

**모바일 앱 (Expo · port 8081)**
- 로그인 (expo-secure-store 토큰 저장)
- 홈/출퇴근 탭: 출퇴근 상태 동적 표시, 30초 갱신
- 출퇴근내역 탭, 스케줄 탭, 급여 탭

**공유 패키지**
- `@workin/types`: User, Store, Schedule, Attendance, Payroll 타입
- `@workin/utils`: `formatKRW`, `formatMinutes`, `calcPayroll`, `calcWeeklyAllowance`, `formatTime`, `getKSTHour`, `getKSTDateStr`

### 🐛 버그 수정 (Bug Fixes)
- KST/UTC 시간 표시 오류 수정: `getHours()` 대신 UTC+9 오프셋 방식(`getUTCHours()`) 사용

## 변경사항 (2차) — 주요 이슈 수정 + 회원가입

### ✨ 새 기능 (Features)

- **웹 회원가입** `/auth/signup`: 이름·이메일·비밀번호·전화번호 폼, 클라이언트 유효성 검사, 가입 후 자동 로그인
- **모바일 회원가입** `(auth)/signup`: 동일 폼, KeyboardAvoidingView(iOS 키보드 대응)
- **공용 Modal 컴포넌트** (`apps/web/src/components/ui/modal.tsx`): ESC 닫기, 백드롭 클릭 닫기, size prop(sm/md/lg)
- **스케줄 시프트 추가 모달**: 직원 선택·날짜·KST 시간 입력 → `POST /stores/:id/schedules`
- **스케줄 블록 삭제**: 블록 클릭 → 확인 모달 → `DELETE /stores/:id/schedules/:sid`
- **알바생 초대 모달**: 초대 코드 발급, 클립보드 복사, 새 코드 재발급
- **급여 직원별 상세 모달**: 행 클릭 → 급여 breakdown + 출퇴근 내역 목록
- **사이드바 `+ 매장 추가` 버튼**: OWNER만 표시, 이름·주소·업종 입력 모달

### 🐛 버그 수정 (Bug Fixes)
- 모바일 KST 시간 표시 버그 4곳 수정 (`formatTime()` 적용):
  - `(tabs)/index.tsx`: 스케줄 시간, 출근 시간
  - `(tabs)/attendance.tsx`: 출퇴근 기록 시간
  - `(tabs)/schedule.tsx`: 주간 스케줄 시간
- 모바일 `attendance.tsx`: 날짜 표시 `locale: undefined` → `ko` 로케일 적용

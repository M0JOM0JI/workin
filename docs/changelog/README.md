# Changelog 작성 규칙

## 파일 네이밍
```
docs/changelog/YYYY-MM-DD.md
```
- 같은 날 여러 번 배포하면 **같은 파일에 섹션을 추가**한다.
- 날짜는 **배포 당일 KST 기준**으로 한다.

## 파일 구조 (템플릿)

```markdown
# YYYY-MM-DD 배포

## 배포 요약
한 줄 요약

## 변경사항

### ✨ 새 기능 (Features)
- ...

### 🐛 버그 수정 (Bug Fixes)
- ...

### ♻️ 리팩터링 (Refactor)
- ...

### 🔒 보안 (Security)
- ...

### 📝 문서 (Docs)
- ...

### 🏗️ 인프라 (Infra)
- ...

## 주요 파일 변경 목록
| 파일 | 변경 내용 |
|------|----------|
| ... | ... |

## 이슈 / 특이사항
- ...
```

## 이모지 기준
| 이모지 | 유형 |
|--------|------|
| ✨ | 새 기능 |
| 🐛 | 버그 수정 |
| ♻️ | 리팩터링 |
| 🔒 | 보안 |
| 📝 | 문서/주석 |
| 🏗️ | 인프라·환경·설정 |
| ⚡ | 성능 개선 |
| 🗑️ | 코드·파일 삭제 |

## 관련 파일
- `CHECKLIST.md` — 기능 구현 현황 (배포와 별개로 항상 최신 유지)
- `CLAUDE.md` — 환경·아키텍처 변경 시 업데이트
- `docs/STARTUP.md` — 기동 절차 변경 시 업데이트

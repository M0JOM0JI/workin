# 알바 관리 서비스 PRD (Product Requirements Document)

> 작성일: 2026-04-20  
> 버전: v0.1

---

## 1. 서비스 개요

### 서비스명
**Workin**

### 한 줄 정의
사장과 알바생을 연결하는 스케줄 · 출근 관리 올인원 플랫폼

### 핵심 가치
- 사장: 다중 매장의 알바생 스케줄을 한눈에 관리
- 알바생: 출근 기록, 급여 확인을 간편하게

---

## 2. 사용자 정의

### 역할 구분

| 역할 | 설명 | 주요 행동 |
|------|------|-----------|
| 오너(Owner) | 브랜드/사업체 최고 관리자 | 매장 생성, 매니저 임명, 전체 현황 조회 |
| 매니저(Manager) | 매장 단위 관리자 | 알바생 등록, 스케줄 작성, 급여 확인 |
| 알바생(Staff) | 근무자 | 출퇴근 체크, 스케줄 확인, 급여 확인 |

### 사용 시나리오

**오너 시나리오**
1. 매장을 생성하고 매니저/알바생을 초대한다
2. 매니저에게 권한을 부여하고 매장 운영을 위임한다
3. 다중 매장 전체 현황(출근, 급여)을 한눈에 파악한다
4. 매장 설정(시급, 매장 정보)을 관리한다

**매니저 시나리오**
1. 오너로부터 초대 코드를 받아 매니저 역할로 합류한다
   (또는 기존 알바생이 오너에게 매니저로 승격된다)
2. 담당 매장의 주간 스케줄을 작성하고 알바생에게 공유한다
3. 실시간 출근 현황을 확인하고 지각/결근을 파악한다
4. 알바생을 초대하고 시급을 등록한다
5. 월말 급여 내역을 확인하고 명세서를 발송한다
6. 단, 매장 삭제 및 매니저 임명은 불가 (오너 전용)

**알바생 시나리오**
1. 초대 링크로 가입 후 매장에 합류한다
2. 오늘의 스케줄을 확인한다
3. 출근/퇴근 버튼을 눌러 기록한다
4. 이번 달 급여 예상액을 확인한다

## 3. 기능 명세

### 3-1. 인증/계정

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 회원가입 | 이메일 또는 소셜(카카오/구글) | P0 |
| 로그인 | 이메일+비밀번호, 소셜 로그인 | P0 |
| 역할 선택 | 가입 시 사장/알바생 선택 | P0 |
| 매장 초대 수락 | 초대 링크/코드로 매장 합류 | P0 |

### 3-2. 매장 관리

| 기능 | 설명 | 오너 | 매니저 | 우선순위 |
|------|------|------|--------|----------|
| 매장 생성 | 매장명, 주소, 업종 등록 | ✅ | ❌ | P0 |
| 매장 편집 | 매장 정보 수정 | ✅ | ❌ | P0 |
| 매장 삭제 | 매장 폐쇄 처리 | ✅ | ❌ | P1 |
| 다중 매장 전환 | 소속 매장 간 빠른 전환 | ✅ | ✅ | P0 |
| 알바생 초대 | 링크 또는 코드로 초대 | ✅ | ✅ | P0 |
| 매니저 초대 | 매니저 역할로 직접 초대 | ✅ | ❌ | P0 |
| 알바생 목록 | 재직/퇴직 필터, 프로필 관리 | ✅ | ✅ | P0 |
| 시급 설정 | 알바생별 시급 등록/수정 | ✅ | ✅ | P0 |
| 매니저 권한 부여 | 알바생 → 매니저 승격 | ✅ | ❌ | P1 |
| 매니저 권한 해제 | 매니저 → 알바생 강등 | ✅ | ❌ | P1 |

### 3-3. 스케줄 관리 (오너/매니저 전용)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 주간 스케줄 작성 | 드래그로 시프트 블록 생성 | P0 |
| 월간 캘린더 뷰 | 매장 전체 스케줄 한눈에 | P0 |
| 반복 스케줄 설정 | 매주 같은 시간 자동 등록 | P1 |
| 스케줄 알림 발송 | 작성 완료 시 알바생에게 푸시 | P1 |
| 스케줄 변경 요청 수락 | 알바생이 요청한 변경 승인/거절 | P2 |

### 3-4. 출근부 (알바생 전용)

| 기능 | 설명 | 우선순위 |

| 급여 명세서 | 알바생별 월간 명세서 생성 | P1 |

## 4. 화면 흐름 (User Flow)

### 오너 플로우
```
앱 실행
  └─ 로그인
       └─ 홈 (매장 대시보드)
            ├─ 스케줄 관리
            │    ├─ 주간 뷰 (시프트 작성/편집)
            │    └─ 월간 뷰
            ├─ 알바생 관리
            │    ├─ 목록
            │    ├─ 상세 (시급 수정 / 매니저 승격)
            │    └─ 초대 (알바생 / 매니저 구분)
            ├─ 출근 현황 (실시간)
            ├─ 급여 관리
            │    ├─ 월별 요약
            │    └─ 알바생별 명세
            └─ 설정
                 ├─ 매장 정보 편집
                 ├─ 새 매장 추가
                 └─ 매장 전환
```

### 매니저 플로우
```
최초 합류
  ├─ [경로 A] 오너가 매니저 초대 코드 발급
  │    └─ 알바생이 코드 입력 → 매니저 역할로 합류
  └─ [경로 B] 기존 알바생 → 오너가 매니저로 승격
       └─ 앱 재실행 시 매니저 UI로 전환

로그인 후
  └─ 홈 (매장 대시보드)          ← 오너 홈과 동일 UI
       ├─ 스케줄 관리
       │    ├─ 주간 뷰 (시프트 작성/편집)
       │    └─ 월간 뷰
       ├─ 알바생 관리
       │    ├─ 목록
       │    ├─ 상세 (시급 수정 가능 / 매니저 승격 불가)
       │    └─ 알바생 초대 (알바생 역할만 가능)
       ├─ 출근 현황 (실시간)
       ├─ 급여 관리
       │    ├─ 월별 요약
       │    └─ 알바생별 명세
       └─ 설정
            └─ 매장 전환 (소속 매장만)
                 ※ 매장 정보 편집 / 삭제 불가
```

### 알바생 플로우
```
앱 실행
  └─ 로그인
       └─ 홈 (오늘 스케줄 + 출근 버튼)
            ├─ 출근 체크 → 퇴근 체크
            ├─ 스케줄 확인 (주간/월간)
            ├─ 출근 기록
            │    └─ 월별 근무 이력
            ├─ 급여 확인
            │    └─ 이번 달 예상 급여
            └─ 설정
```

## 6. 데이터 모델 (초안)

```
User (사용자)
  ├─ id, email, name, phone, role
  └─ created_at

Store (매장)
  ├─ id, owner_id, name, address, brand
  └─ created_at

StoreStaff (매장-직원 관계)
  ├─ store_id, user_id, role(manager/staff), hourly_wage
  └─ joined_at, left_at

Schedule (스케줄)
  ├─ id, store_id, staff_id
  ├─ start_at, end_at
  └─ is_recurring, repeat_rule

Attendance (출퇴근 기록)
  ├─ id, store_id, staff_id, schedule_id
  ├─ clock_in, clock_out
  └─ memo, location(lat/lng)

Payroll (급여)
  ├─ id, store_id, staff_id, year_month
  ├─ total_hours, base_pay, deduction, net_pay
  └─ is_confirmed
```

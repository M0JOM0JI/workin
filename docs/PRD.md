# 알바 관리 서비스 PRD (Product Requirements Document)

> 작성일: 2026-04-20 | 최종 수정: 2026-04-22  
> 버전: v0.2

---

## 1. 서비스 개요

### 서비스명
**Workin**

### 한 줄 정의
사장과 알바생을 연결하는 스케줄 · 출근 관리 올인원 플랫폼

### 핵심 가치
- 사장: 다중 매장의 알바생 스케줄을 한눈에 관리
- 알바생: 출근 기록, 급여 확인을 간편하게

---

## 2. 사용자 정의

### 역할 구분

| 역할 | 설명 | 주요 행동 |
|------|------|-----------|
| 오너(Owner) | 브랜드/사업체 최고 관리자 | 매장 생성, 매니저 임명, 전체 현황 조회 |
| 매니저(Manager) | 매장 단위 관리자 | 알바생 등록, 스케줄 작성, 급여 확인 |
| 알바생(Staff) | 근무자 | 출퇴근 체크, 스케줄 확인, 급여 확인 |

### 사용 시나리오

**오너 시나리오**
1. 매장을 생성하고 매니저/알바생을 초대한다
2. 매니저에게 권한을 부여하고 매장 운영을 위임한다
3. 다중 매장 전체 현황(출근, 급여)을 한눈에 파악한다
4. 매장 설정(시급, 매장 정보)을 관리한다

**매니저 시나리오**
1. 오너로부터 초대 코드를 받아 매니저 역할로 합류한다
   (또는 기존 알바생이 오너에게 매니저로 승격된다)
2. 담당 매장의 주간 스케줄을 작성하고 알바생에게 공유한다
3. 실시간 출근 현황을 확인하고 지각/결근을 파악한다
4. 알바생을 초대하고 시급을 등록한다
5. 월말 급여 내역을 확인하고 명세서를 발송한다
6. 단, 매장 삭제 및 매니저 임명은 불가 (오너 전용)

**알바생 시나리오**
1. 초대 링크로 가입 후 매장에 합류한다
2. 오늘의 스케줄을 확인한다
3. 출근/퇴근 버튼을 눌러 기록한다
4. 이번 달 급여 예상액을 확인한다

| 매장 삭제 | 매장 폐쇄 처리 | ✅ | ❌ | P1 |
| 다중 매장 전환 | 소속 매장 간 빠른 전환 (사이드바 셀렉트) | ✅ | ✅ | P0 |
| 알바생 초대 | 링크 또는 코드로 초대 | ✅ | ✅ | P0 |
| 매니저 초대 | 매니저 역할로 직접 초대 | ✅ | ❌ | P0 |
| 알바생 목록 | 재직/퇴직 필터, 프로필 관리 | ✅ | ✅ | P0 |
| 시급 설정 | 알바생별 시급 등록/수정 | ✅ | ✅ | P0 |
| 매니저 권한 부여 | 알바생 → 매니저 승격 | ✅ | ❌ | P1 |

## 4. 화면 흐름 (User Flow)

### 오너 플로우
```
앱 실행
  └─ 로그인
       ├─ [매장 없음] → 온보딩: 매장 생성 화면 (필수)
       │                  └─ 매장명·사업자명·사업자번호·주소 등 입력
       │                       └─ 생성 완료 → 홈 (매장 대시보드)
       │
       └─ [매장 있음] → 홈 (매장 대시보드)
            ├─ 사이드바: [+ 매장 추가] 버튼 (항상 노출)
            │              └─ 매장 생성 모달 (동일 입력 폼)
            ├─ 스케줄 관리
            │    ├─ 주간 뷰 (시프트 작성/편집)
            │    └─ 월간 뷰
            ├─ 알바생 관리
            │    ├─ 목록
            │    ├─ 상세 (시급 수정 / 매니저 승격)
            │    └─ 초대 (알바생 / 매니저 구분)
            ├─ 출근 현황 (실시간)
            ├─ 급여 관리
            │    ├─ 월별 요약
            │    └─ 알바생별 명세
            └─ 설정
                 ├─ 매장 정보 편집 (확장 필드 포함)
                 └─ 매장 전환
```

### 매니저 플로우
```
최초 합류
  ├─ [경로 A] 오너가 매니저 초대 코드 발급
  │    └─ 알바생이 코드 입력 → 매니저 역할로 합류
  └─ [경로 B] 기존 알바생 → 오너가 매니저로 승격
       └─ 앱 재실행 시 매니저 UI로 전환

로그인 후
  └─ 홈 (매장 대시보드)          ← 오너 홈과 동일 UI
       ├─ 스케줄 관리
       │    ├─ 주간 뷰 (시프트 작성/편집)
       │    └─ 월간 뷰
       ├─ 알바생 관리
       │    ├─ 목록
       │    ├─ 상세 (시급 수정 가능 / 매니저 승격 불가)
       │    └─ 알바생 초대 (알바생 역할만 가능)
       ├─ 출근 현황 (실시간)
       ├─ 급여 관리
       │    ├─ 월별 요약
       │    └─ 알바생별 명세
       └─ 설정
            └─ 매장 전환 (소속 매장만)
                 ※ 매장 정보 편집 / 삭제 불가
```

### 알바생 플로우
```
앱 실행
  └─ 로그인
       └─ 홈 (오늘 스케줄 + 출근 버튼)
            ├─ 출근 체크 → 퇴근 체크
            ├─ 스케줄 확인 (주간/월간)
            ├─ 출근 기록
            │    └─ 월별 근무 이력
            ├─ 급여 확인
            │    └─ 이번 달 예상 급여
            └─ 설정
```

## 5. 플랫폼 & 기술 방향

### 지원 플랫폼
- **웹**: PC 브라우저 (사장 중심, 스케줄 작성 최적화)
- **모바일 앱**: iOS / Android — **하이브리드 방식** (웹 개발 완료 후 진행)

### 모바일 앱 전략 — 하이브리드 (WebView)

> **진행 시점**: 웹·API 전체 기능 완성 후 착수

웹 앱(Next.js)을 Expo WebView로 래핑해 Android / iOS 앱을 배포한다.  
별도 네이티브 코드베이스를 유지하지 않으며, 웹 기능 추가 시 앱도 자동 반영된다.

```
Next.js Web App (반응형 + PWA)
        ↓  Expo WebView 래핑
  ┌─────────────┐    ┌─────────────┐
  │  Android    │    │    iOS      │
  │  (APK/AAB)  │    │   (IPA)     │
  │ Google Play │    │  App Store  │
  └─────────────┘    └─────────────┘
        공통: 푸시 알림, 딥링크, 아이콘/스플래시
```

#### Android 앱
| 항목 | 내용 |
|------|------|
| 빌드 도구 | EAS Build (Expo Application Services) |
| 배포 형식 | APK (테스트) / AAB (Play Store) |
| 스토어 | Google Play Console |
| 계정 | Google Play 개발자 계정 ($25 1회) |
| 최소 지원 OS | Android 8.0 (API 26) 이상 |
| 핵심 네이티브 기능 | 푸시 알림 (FCM), 딥링크 |

#### iOS 앱
| 항목 | 내용 |
|------|------|
| 빌드 도구 | EAS Build |
| 배포 형식 | IPA → TestFlight (테스트) / App Store |
| 스토어 | Apple App Store Connect |
| 계정 | Apple Developer Program ($99/년) |
| 최소 지원 OS | iOS 15 이상 |
| 핵심 네이티브 기능 | 푸시 알림 (APNs), 딥링크 |

#### 공통 작업 목록
| 단계 | 작업 |
|------|------|
| 1. 웹 모바일 최적화 | 반응형 UI 점검, 터치 이벤트, 뷰포트 설정, PWA manifest |
| 2. Expo Shell 구성 | WebView 컴포넌트로 웹 URL 로드, 토큰 Bridge(웹↔네이티브) |
| 3. 앱 정체성 | 아이콘(1024×1024), 스플래시 화면, 앱 이름·번들 ID |
| 4. 푸시 알림 | Expo Notifications, FCM(Android), APNs(iOS) 키 등록 |
| 5. 딥링크 | 초대 코드 링크 클릭 → 앱 내 해당 화면 이동 |
| 6. 빌드 & 테스트 | EAS Build 로컬/클라우드, 실기기 테스트 |
| 7. 스토어 등록 | 스크린샷, 설명, 개인정보 처리방침, 심사 제출 |

### 권장 기술 스택 (확정 전 검토 필요)

| 영역 | 후보 | 비고 |
|------|------|------|
| 프론트엔드 웹 | Next.js (React) | SEO + SSR 지원 |
| 모바일 앱 | React Native (Expo) | 웹과 코드 공유 |
| 백엔드 API | NestJS (Node.js) | TypeScript 통일 |
| 데이터베이스 | PostgreSQL | 다중 매장 관계형 구조 |
| 인증 | Supabase Auth 또는 자체 JWT | |
| 푸시 알림 | Firebase Cloud Messaging | |
| 배포 | Vercel (웹) + AWS/Railway (백엔드) | |

## 6. 데이터 모델 (초안)

```
User (사용자)
  ├─ id, email, name, phone, role
  └─ created_at

Store (매장)
  ├─ id, owner_id
  ├─ name            (매장명, 필수)
  ├─ businessOwner   (사업자명, 필수)
  ├─ businessNumber  (사업자번호 000-00-00000, 필수)
  ├─ address         (주소, 필수)
  ├─ phone           (매장 전화번호, 선택)
  ├─ mobilePhone     (담당자 휴대폰번호, 선택)
  ├─ category        (업종, 선택)
  └─ created_at

StoreStaff (매장-직원 관계)
  ├─ store_id, user_id, role(manager/staff), hourly_wage
  └─ joined_at, left_at

Schedule (스케줄)
  ├─ id, store_id, staff_id
  ├─ start_at, end_at
  └─ is_recurring, repeat_rule

Attendance (출퇴근 기록)
  ├─ id, store_id, staff_id, schedule_id
  ├─ clock_in, clock_out
  └─ memo, location(lat/lng)

Payroll (급여)
  ├─ id, store_id, staff_id, year_month
  ├─ total_hours, base_pay, deduction, net_pay
  └─ is_confirmed
```

## 1. 서비스 개요

### 서비스명
**Workin**

### 한 줄 정의
사장과 알바생을 연결하는 스케줄 · 출근 관리 올인원 플랫폼

### 핵심 가치
- 사장: 다중 매장의 알바생 스케줄을 한눈에 관리
- 알바생: 출근 기록, 급여 확인을 간편하게
- 공통: 투명한 출퇴근 이력으로 노사 간 신뢰 확보

## 2. 사용자 정의

### 역할 구분

| 역할 | 설명 | 주요 행동 |
|------|------|-----------|
| 오너(Owner) | 브랜드/사업체 최고 관리자 | 매장 생성, 매니저 임명, 전체 현황 조회 |
| 매니저(Manager) | 매장 단위 관리자 | 알바생 등록, 스케줄 작성, 급여 확인 |
| 알바생(Staff) | 근무자 | 출퇴근 체크, 스케줄 확인, 급여 확인 |

### 사용 시나리오

**오너 시나리오**
1. 매장을 생성하고 매니저/알바생을 초대한다 (역할 구분 초대 코드 발급)
2. 매니저에게 권한을 부여하고 매장 운영을 위임한다 (승격/강등)
3. 다중 매장 전체 현황(출근, 급여)을 한눈에 파악한다
4. 매장 설정(시급, 야간수당, 자동퇴근, 급여일 등)을 관리한다
5. 알바생의 출퇴근·스케줄 수정 요청을 승인/거절하거나 직접 등록한다
6. 직원별 근무 통계(지각·결근·평균 근무시간)로 운영을 분석한다

**매니저 시나리오**
1. 오너로부터 매니저 초대 코드를 받아 합류한다
2. 담당 매장의 주간 스케줄을 작성하고 알바생에게 확정 알림을 보낸다
3. 알바생의 근무 희망 시간을 참고해 효율적으로 스케줄을 배정한다
4. 실시간 출근 현황을 확인하고 지각/결근/초과근무를 파악한다
5. 알바생의 대타 요청·스케줄 변경 요청·출퇴근 수정 요청을 처리한다
6. 월말 급여 내역을 확인하고 확정 처리한다

**알바생 시나리오**
1. 초대 코드로 가입 후 매장에 합류한다
2. 근무 희망 시간을 등록해 스케줄 배정에 반영 요청한다
3. 오늘의 스케줄을 확인하고 출근/퇴근 버튼을 누른다
4. 퇴근 누락 시 출퇴근 수정 요청을 제출한다
5. 불가피한 상황 시 스케줄 변경 요청 또는 대타 요청을 올린다
6. 이번 달 급여 예상액(야간수당·주휴수당·공제 포함)을 확인한다

## 4. 화면 흐름 (User Flow)

### 오너 플로우
```
앱 실행
  └─ 로그인
       ├─ [매장 없음] → 온보딩: 매장 생성 화면 (필수)
       │                  └─ 매장명·사업자명·사업자번호·주소 등 입력
       │                       └─ 생성 완료 → 홈 (매장 대시보드)
       │
       └─ [매장 있음] → 홈 (매장 대시보드)
            ├─ 사이드바: [+ 매장 추가] 버튼 (OWNER만)
            ├─ 스케줄 관리
            │    ├─ 주간 뷰 (시프트 작성/수정/삭제/확정)
            │    ├─ 월간 뷰 (달력, 도트 표시)
            │    ├─ 주간 복사
            │    └─ 근무 희망 시간 참고 표시
            ├─ 알바생 관리
            │    ├─ 목록 (이름 검색, 재직/퇴직 필터, 계약시간·통계)
            │    ├─ 상세 (시급·4대보험·계약시간·메모 / 매니저 승격/강등 / 퇴직)
            │    └─ 초대 (STAFF 코드 / MANAGER 코드 구분 발급)
            ├─ 출근 현황 (실시간, 지각·초과근무 배지)
            │    └─ 수정 요청 탭 (승인/거절, 이력)
            ├─ 급여 관리
            │    ├─ 월별 요약 + 확정 버튼
            │    └─ 알바생별 명세 (야간수당·주휴수당·공제·4대보험 포함)
            ├─ 통계 (직원별 근무 통계, 인건비 비율)
            └─ 설정
                 ├─ 매장 기본 정보 편집
                 ├─ 자동 퇴근 설정
                 ├─ 야간수당·초과수당 설정
                 └─ 급여일 설정
```

### 매니저 플로우
```
최초 합류
  ├─ [경로 A] 오너가 MANAGER 초대 코드 발급 → 코드 입력 → 매니저로 합류
  └─ [경로 B] 기존 알바생 → 오너가 매니저로 승격

로그인 후 (오너와 동일 UI, 권한 제한만 다름)
  ├─ 스케줄 관리 (작성/수정/확정 가능)
  ├─ 알바생 관리 (STAFF 초대만, 승격/강등 불가)
  ├─ 출근 현황 + 수정 요청 처리
  ├─ 급여 관리 (확인·확정 가능)
  ├─ 통계 (직원별 근무 통계, 인건비 비율 입력)
  └─ 설정 (읽기 전용 — 매장 정보·야간수당·급여일 편집 불가)
```

### 알바생 플로우
```
앱 실행 (모바일)
  └─ 로그인
       └─ 홈 (공지사항 배너 + 오늘 스케줄 + 출근 버튼)
            ├─ 출근 체크 → 퇴근 체크
            ├─ 스케줄 확인
            │    ├─ 주간 목록 (총 근무시간 합계)
            │    ├─ 스케줄 변경 요청 제출
            │    └─ 대타 요청 올리기
            ├─ 출근 기록
            │    ├─ 월별 근무 이력 (자동퇴근 배지, 지각 배지)
            │    ├─ 수정 요청 제출 (사유 + 희망 시간)
            │    └─ 요청 상태 확인 (대기중/승인/거절 + 이력)
            ├─ 급여 확인
            │    └─ 이번 달 예상 급여 (야간수당·주휴수당·공제 breakdown)
            │         급여일 D-N 표시
            └─ 프로필/설정
                 ├─ 이름·전화번호 수정
                 ├─ 근무 희망 시간 등록 (요일별)
                 └─ 로그아웃
```

## 6. 데이터 모델

```
User (사용자)
  ├─ id, email, name, phone, passwordHash
  └─ createdAt

Store (매장)
  ├─ id, name, address, category
  ├─ businessOwner, businessNumber
  ├─ phone, mobilePhone
  ├─ payDay (Int, 1~31)              ← 급여일
  ├─ autoClockOut (bool)             ← 자동퇴근 활성화
  ├─ autoClockOutMode                ← SCHEDULE | MAX_HOURS | MIDNIGHT
  ├─ autoClockOutBuffer (분)         ← SCHEDULE 모드 버퍼
  ├─ autoClockOutMaxHours (시간)     ← MAX_HOURS 모드 기준
  ├─ nightShiftEnabled (bool)        ← 야간수당 활성화
  ├─ nightShiftMultiplier (Float)    ← 야간수당 배율 (기본 1.5)
  ├─ overtimeEnabled (bool)          ← 초과근무 수당 활성화
  ├─ overtimeMultiplier (Float)      ← 초과근무 배율
  └─ createdAt

StoreStaff (매장-직원 관계)
  ├─ storeId, userId
  ├─ role (OWNER | MANAGER | STAFF)
  ├─ hourlyWage
  ├─ contractHoursPerMonth (Int?)    ← 월간 계약 근무시간
  ├─ insuranceType                   ← NONE | FOUR_MAJOR | THREE_THREE
  ├─ memo (String?)                  ← 내부 메모 (오너/매니저만 열람)
  └─ joinedAt, leftAt

WorkPreference (근무 희망 시간)
  ├─ id, storeId, staffId
  ├─ dayOfWeek (0=일~6=토)
  ├─ startTime, endTime
  └─ createdAt

Schedule (스케줄)
  ├─ id, storeId, staffId
  ├─ startAt, endAt
  ├─ isConfirmed (bool)              ← 스케줄 확정 여부
  └─ isRecurring, repeatRule

Attendance (출퇴근 기록)
  ├─ id, storeId, staffId, scheduleId
  ├─ clockIn, clockOut
  ├─ memo
  ├─ isAutoClockOut (bool)           ← 자동퇴근 여부
  └─ location (lat/lng)

AttendanceRequest (출퇴근 수정 요청)
  ├─ id, storeId, staffId
  ├─ requestedBy (요청자 userId)
  ├─ attendanceId (기존 수정 시 연결, 신규면 null)
  ├─ requestedClockIn, requestedClockOut
  ├─ reason
  ├─ status (PENDING | APPROVED | REJECTED)
  ├─ reviewedBy, reviewedAt, reviewNote
  └─ createdAt

ScheduleRequest (스케줄 변경 요청)
  ├─ id, storeId, staffId
  ├─ scheduleId (대상 스케줄)
  ├─ requestType (CHANGE | CANCEL | SUBSTITUTE)
  ├─ requestedStartAt, requestedEndAt
  ├─ reason
  ├─ status (PENDING | APPROVED | REJECTED)
  ├─ reviewedBy, reviewedAt, reviewNote
  └─ createdAt

Notification (인앱 알림)
  ├─ id, userId, storeId
  ├─ type (ATTENDANCE_REQUEST | SCHEDULE_CONFIRMED | PAYROLL_CONFIRMED | ...)
  ├─ title, body
  ├─ referenceId (연관 레코드 ID)
  ├─ isRead (bool)
  └─ createdAt

Announcement (공지사항)
  ├─ id, storeId, authorId
  ├─ title, content
  ├─ isPinned (bool)
  └─ createdAt

Payroll (급여)
  ├─ id, storeId, staffId, yearMonth
  ├─ totalMinutes, regularMinutes, nightMinutes, overtimeMinutes
  ├─ basePay, nightAllowance, overtimePay
  ├─ weeklyAllowance, deduction, netPay
  └─ isConfirmed
```
